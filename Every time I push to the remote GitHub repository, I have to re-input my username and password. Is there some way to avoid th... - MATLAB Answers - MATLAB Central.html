Is there some way to avoid this by having MATLAB store this information? When I use SourceTree to manage my Git repositories the account info is stored, but then SourceTree is not embedded in MATLAB. I would like to have the best of both.

 Accepted Answer

The best way to do this is with SSH key pairs to validate that a given computer is allowed to interact with the remote repository. As long as the SSH key generated doesn’t have a password, MATLAB will no longer ask for credentials when pushing or pulling code from remote repositories accessible by that GitHub account. When working with repositories hosted by GitHub, there is a specific workflow recommended for the SSH key pair generation and verification:
1) Open Git Bash or other "git" command window.
2) Generate the ssh-key pair:
ssh-keygen -t rsa -m PEM -b 4096 -C "email_address@example.com"
Where the provided email address is used as an identifier within the public key. Save the key to the default location and do not create a password.
3) Enable ssh-agent and add the newly-created SSH key:
eval $(ssh-agent -s)
ssh-add ~/.ssh/id_rsa
If this doesn't work, try:
ssh-agent -s
ssh-add ~/.ssh/id_rsa
4) If this is successful, it will generate an ASCII image. Copy the SSH public key to the clip board:
clip < ~/.ssh/id_rsa.pub
5) Associate the public key with a GitHub account by logging on to www.github.com and navigating to Settings>SSH keys>Add SSH key. In the Title field, add a descriptive label for the computer the SSH key is associated with. In the Key field, paste the key that is copied to the clipboard with ctrl+v or cmd+v or by right clicking.
6) Click "Add key". This can be validated by returning to the Git Bash console and executing:
ssh -T git@github.com
7) If there is not a local copy of the repository of interest, clone the repository via SSH:
git clone git@github.com:username/repository
Where 'username' and 'repository' are the GitHub account and code bank in question. If this fails, the repository may not be visible to the GitHub account associated with the generated SSH key or "git" configuration may be incorrect. Repositories can also be cloned from within MATLAB by right clicking in the white space within the Current Folder window and navigating to Source Control>Manage Files... Then, in the "Source control integration" field select "git" and specify the "Repository path" field as "git@github.com:username/repository". In the "Sandbox" field, enter the destination where the repository should be cloned. This should be a new folder, as MATLAB will not allow you to specify a directory that already has contents. Press the "Retrieve" button.
8) If there is already a local copy of the repository of interest, navigate to it in Git Bash and execute:
git remote -v
To determine the remote URL git is currently using. This should look like:
origin https://github.com:username/repository.git (fetch)
origin https://github.com:username/repository.git (push)
9) Change the remote URL to the following:
git remote set-url origin git@github.com:username/repository.git
Where 'username' and 'repository' are the GitHub account and code bank in question.
10) Verify the URL has changed using:
git remote -v
11) If MATLAB is open, restart it for the changes to take effect. Now when attempting to push or pull from within MATLAB, the user should no longer be asked for a password.
